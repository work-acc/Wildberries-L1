Чем отличаются буферизированные и не буферизированные каналы?

Ответ:
```
Не буферизированные каналы:
При создании не буферизированного канала используется синтаксис 
make(chan T), где T - тип данных, который будет передаваться через 
канал.
Когда горутина отправляет значение в не буферизированный канал 
с помощью оператора <-, отправка блокируется, пока другая горутина 
не прочитает это значение из канала.
Аналогично, когда горутина пытается прочитать значение из 
не буферизированного канала, чтение блокируется, пока другая 
горутина не отправит значение в канал.
Не буферизированные каналы обеспечивают синхронизацию между 
горутинами и гарантируют, что отправка и получение данных 
происходит в определенном порядке.

Буферизированные каналы:
При создании буферизированного канала используется синтаксис 
make(chan T, capacity), где T - тип данных, а capacity 
- размер буфера канала (количество элементов, которые могут 
быть помещены в канал без блокировки).
Отправка данных в буферизированный канал с помощью оператора 
<- не блокирует выполнение горутины, если буфер канала не заполнен.
Чтение данных из буферизированного канала также не блокирует 
выполнение горутины, если буфер не пуст.
Если буфер канала заполнен и попытка отправить значение, 
отправка блокируется до момента освобождения места в буфере, 
аналогично, если буфер канала пуст и попытка чтения значения, 
чтение блокируется до момента отправки значения в канал.
```
